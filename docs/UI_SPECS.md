# GAR UI Specs

## 공통 원칙

- 모든 페이지는 `loading`, `empty`, `error` 상태를 가진다.
- 필터는 URL query string에 반영되어 새로고침 후 유지된다.
- 분석 상태가 미완료면 skeleton 또는 상태 배지를 보여준다.
- `needs_review` 데이터는 경고 아이콘과 함께 노출한다.

## 1. Dashboard

### 목적

- 오늘의 새 공고와 우선 확인해야 할 공고를 빠르게 보여준다.

### 구성

- 오늘 새 공고 수
- 마감 임박 공고
- 매칭 상위 공고
- 최근 실패 작업 알림

### Acceptance Criteria

- 첫 진입 3초 내 핵심 숫자 카드가 보여야 한다.
- 공고 카드 클릭 시 상세로 이동한다.
- 데이터가 없어도 빈 상태 문구가 있어야 한다.

## 2. Bids Index

### 목적

- 사용자가 공고를 탐색하고 필터링한다.

### 필터

- 지역
- 차종
- 예산 범위
- 마감일
- 분류코드
- 분석 상태
- 매칭 점수 범위

### Acceptance Criteria

- 필터 변경 시 목록이 갱신된다.
- 정렬은 최신순/마감순/금액순/매칭순을 지원한다.
- 카드뷰와 리스트뷰 전환을 지원한다.

## 3. Bid Detail

### 목적

- 한 공고의 원문과 구조화 정보를 함께 본다.

### 구성 (정보 우선순위 순서)

1. **자격 체크** — "이거 나 되냐" (소재지/업종/기업규모 기준, 프로필 대비 ✅/❌)
2. **차량 정보** — 뭘 원하는지 (차종/대수/트림/연식/옵션)
3. **예산/마감** — 얼마짜리, 언제까지
4. **계약 조건** — 기간/선납금/주행거리/보험
5. **서류 체크리스트** — 뭘 준비해야 하는지
6. **적격심사 시뮬레이션** — 프로필 기반 예상 점수 (프로필 완성 시)
7. 원본 첨부파일 목록

### Acceptance Criteria

- 분석 전에는 "분석 대기" 상태가 보인다.
- 분석 실패 시 실패 이유가 최소한 한 줄로 표시된다.
- 첨부파일 다운로드 링크가 제공된다.

## 4. Company Profile

### 목적

- 회사 정보와 기업규모를 관리한다.

### Acceptance Criteria

- 본사와 지점을 구분해 저장할 수 있다.
- 기업규모와 업종코드를 저장할 수 있다.

## 5. Fleet / Preferences

### 목적

- 보유 차종과 관심 조건을 설정한다.

### Acceptance Criteria

- 차종/대수 추가, 수정, 삭제가 가능하다.
- 관심 지역/예산/기간/채널을 저장할 수 있다.
- 저장 후 다음 매칭부터 반영된다.

## 6. Simulation

### 목적

- 공고별 적격심사 예상 점수를 프로필 기반으로 시뮬레이션한다.

### 구성

- 심사분야별 배점/예상점수 테이블
- 부족 항목 강조 + "프로필에서 이 정보를 추가하면 정확도가 올라갑니다"
- 프로필 미완성 시 "정보 부족" 표시 (0점이 아닌 unknown)

### Acceptance Criteria

- 프로필 입력 후 공고 상세에서 시뮬레이션 결과가 보인다.
- 프로필 정보가 부족하면 부분 결과 + 안내가 보인다.
- "85점 넘으려면 투찰률 몇 % 이상" 가이드 표시.

## 7. 공통 권한

- 일반 사용자: 공고 열람, 알림 확인, 자격 체크, 시뮬레이션 확인
- 관리자: 회사 정보, 프로필, 선호도, 사용자 관리

## 8. 공통 오류 메시지 기준

- 외부 수집 실패: "현재 최신 공고 동기화가 지연되고 있습니다."
- 분석 실패: "첨부문서 분석에 실패했습니다. 원문을 확인해 주세요."
- 시뮬레이션 부분: "프로필 정보가 부족해 일부 항목을 계산하지 못했습니다."
