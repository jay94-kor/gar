# GAR UI Specs

## 공통 원칙

- 모든 페이지는 `loading`, `empty`, `error` 상태를 가진다.
- 필터는 URL query string에 반영되어 새로고침 후 유지된다.
- 분석 상태가 미완료면 skeleton 또는 상태 배지를 보여준다.
- `needs_review` 데이터는 경고 아이콘과 함께 노출한다.

## 1. Dashboard

### 목적

- 오늘의 새 공고와 우선 확인해야 할 공고를 빠르게 보여준다.

### 구성

- 오늘 새 공고 수
- 마감 임박 공고
- 매칭 상위 공고
- 최근 실패 작업 알림

### Acceptance Criteria

- 첫 진입 3초 내 핵심 숫자 카드가 보여야 한다.
- 공고 카드 클릭 시 상세로 이동한다.
- 데이터가 없어도 빈 상태 문구가 있어야 한다.

## 2. Bids Index

### 목적

- 사용자가 공고를 탐색하고 필터링한다.

### 필터

- 지역
- 차종
- 예산 범위
- 마감일
- 분류코드
- 분석 상태
- 매칭 점수 범위

### Acceptance Criteria

- 필터 변경 시 목록이 갱신된다.
- 정렬은 최신순/마감순/금액순/매칭순을 지원한다.
- 카드뷰와 리스트뷰 전환을 지원한다.

## 3. Bid Detail

### 목적

- 한 공고의 원문과 구조화 정보를 함께 본다.

### 구성

- 공고 헤더
- AI 요약 카드
- 차량/계약/보험/자격/평가 섹션
- 원본 첨부파일 목록
- 매칭 결과
- 견적 생성 CTA

### Acceptance Criteria

- 분석 전에는 "분석 대기" 상태가 보인다.
- 분석 실패 시 실패 이유가 최소한 한 줄로 표시된다.
- 첨부파일 다운로드 링크가 제공된다.

## 4. Company Profile

### 목적

- 회사 정보와 기업규모를 관리한다.

### Acceptance Criteria

- 본사와 지점을 구분해 저장할 수 있다.
- 기업규모와 업종코드를 저장할 수 있다.

## 5. Fleet / Preferences

### 목적

- 보유 차종과 관심 조건을 설정한다.

### Acceptance Criteria

- 차종/대수 추가, 수정, 삭제가 가능하다.
- 관심 지역/예산/기간/채널을 저장할 수 있다.
- 저장 후 다음 매칭부터 반영된다.

## 6. Price Table

### 목적

- 견적 생성에 필요한 가격 데이터를 관리한다.

### Acceptance Criteria

- 모델, 계약개월, 주행거리 구간 기준 단가를 저장할 수 있다.
- 보험/옵션 가산 금액을 별도 입력할 수 있다.
- 누락 필드가 있으면 저장 전에 경고를 표시한다.

## 7. Estimate Builder / Estimate Detail

### 목적

- 공고와 단가표를 바탕으로 견적 초안을 생성하고 검토한다.

### Acceptance Criteria

- 견적 생성 후 line item과 총액이 분리 표시된다.
- 수동 검토가 필요한 항목은 강조된다.
- PDF 다운로드 버튼이 있다.

## 8. 공통 권한

- 일반 사용자: 공고 열람, 알림 확인, 견적 생성
- 관리자: 회사 정보, 단가표, 선호도, 사용자 관리

## 9. 공통 오류 메시지 기준

- 외부 수집 실패: "현재 최신 공고 동기화가 지연되고 있습니다."
- 분석 실패: "첨부문서 분석에 실패했습니다. 원문을 확인해 주세요."
- 견적 불완전: "단가표 정보가 부족해 수동 검토가 필요합니다."
