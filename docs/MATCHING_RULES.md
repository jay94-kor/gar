# GAR Matching Rules

## 목적

이 문서는 공고와 고객사 사이의 적합도 계산, 알림 기준, 설명 가능성 규칙을 정의한다.

## 1. 입력 데이터

- `Bid`
- 최신 `validated` 또는 `needs_review` 분석 결과
- `Company`
- `CompanyFleet`
- `CompanyPreference`

## 2. 평가 순서

1. 하드 필터 적용
2. 하드 필터 통과 시 점수 계산
3. 임계값 이상이면 즉시 알림 큐 적재
4. 임계값 미만이면 목록 노출만 수행

## 3. 하드 필터

아래 중 하나라도 불충족이면 `ineligible`다.

- 업종코드 요건 미충족
- 지역 제한이 있는데 회사 본사/지점이 모두 미충족
- 공동도급 금지인데 단독 자격 미충족
- 필수 차종이 아예 보유 fleet와 교집합 없음
- 지점/정비 네트워크 요구를 전혀 충족하지 못함

## 4. 점수 계산

총점 100점.

| 항목 | 배점 | 계산 규칙 |
|------|------|-----------|
| 지역 매칭 | 30 | 본사/지점이 요구 지역과 일치하면 30, 인접 권역이면 15, 아니면 0 |
| 차종 매칭 | 25 | 요구 차종 전부 보유 25, 일부 보유 10~20, 미보유 0 |
| 규모 매칭 | 20 | 예산이 선호 범위 내 20, 근접 범위 10, 벗어나면 0 |
| 자격 매칭 | 15 | 기업규모/업종/기타 자격 충족률 기반 0~15 |
| 기간 매칭 | 10 | 선호 계약기간과 정확히 일치 10, 근접 5, 아니면 0 |

## 5. 알림 기준

- `score >= 70`: 즉시 알림 대상
- `50 <= score < 70`: 다이제스트 포함 가능
- `score < 50`: 목록 노출만
- `ineligible`: 알림 제외

## 6. 설명 가능한 출력

매칭 결과에는 아래 필드를 함께 저장한다.

```json
{
  "eligibility": "eligible",
  "score_total": 82,
  "breakdown": {
    "region": 30,
    "vehicle": 20,
    "budget": 12,
    "qualify": 10,
    "period": 10
  },
  "reasons": [
    "충남 지역 제한 충족",
    "스타리아 2대 보유",
    "예산 범위 일부 초과"
  ]
}
```

## 7. 중복 발송 방지

- 즉시 알림은 `(company_id, bid_id, channel, notification_type)` 조합으로 한 번만 발송
- 점수가 변해도 같은 채널의 같은 유형 알림은 재발송하지 않음
- 단, 마감 임박 알림은 별도 `notification_type=deadline`으로 분리

## 8. 튜닝 원칙

- 가중치 변경은 `matching_version`을 올린다
- 점수 조정 전후로 과거 공고 50건 이상에 대한 회귀 비교를 수행한다
- 고객 피드백으로 매칭 품질이 낮은 경우, 가중치보다 하드 필터를 먼저 수정한다

## 9. 예외 규칙

- 분석 상태가 `needs_review`면 점수는 계산하되 `confidence_badge=warning`
- 차량 정보가 불완전하면 차종 점수는 보수적으로 계산한다
- 예산 정보가 없으면 규모 매칭은 0점이 아니라 `unknown` 처리 후 총점 정규화 가능 여부를 추후 검토한다
